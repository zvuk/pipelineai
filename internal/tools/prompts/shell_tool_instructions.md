# `shell` — интерфейс выполнения кратких терминальных команд

## 1. Назначение
`shell` запускает короткие, неинтерактивные команды в терминале. Интерфейс рассчитан на диагностику, навигацию и стандартные проверки.

## 2. Разрешённые классы задач
`shell` применим для:
- поиска по репозиторию и чтения файлов,
- сборки/тестов,
- диагностики окружения,
- подготовки артефактов (при большом выводе — запись в файл с последующим кратким резюме).

## 3. Режим безопасности (обязательный)
Поведение по умолчанию:
- выбирать команды, которые не меняют состояние системы (чтение, поиск, статический анализ).

Ограничения:
- разрушительные действия (удаление, push, переписывание истории, рискованные системные команды) допускаются только при явном разрешении в инструкциях/политике.
- любые команды должны быть исполнимы unattended (без интерактивных подтверждений), как в CI.

## 4. Паттерны работы с репозиторием
- основной инструмент поиска: `rg` (ripgrep);
- вывод ограничивать сразу: `head`, `sed -n`;
- большие файлы читать диапазонами:
    - `sed -n '1,250p' file`, затем при необходимости `sed -n '251,500p' file`;
- избегать «простыней» в чате: результат сохранять в файл/артефакт и давать выжимку.

## 5. Схема запроса (объект JSON)
Передаётся JSON со следующими полями:
- `command`: массив строк (как правило `["bash","-lc","..."]`);
- `workdir`: опционально, рабочая директория;
- `timeout_ms`: опционально, таймаут выполнения в миллисекундах.

## 6. Примеры вызовов

Простейшая команда:
```text
shell {"command":["bash","-lc","echo hello"]}
````

Список файлов в каталоге + таймаут:

```text
shell {"command":["bash","-lc","ls -la docs"],"workdir":".","timeout_ms":2000}
```

Поиск `TODO|FIXME` и ограничение выдачи:

```text
shell {"command":["bash","-lc","rg -n \"TODO|FIXME\" -S | head -n 100"]}
```

Поиск файлов с совпадениями по regex и агрегация по путям:

```text
shell {"command":["bash","-lc","rg -n --no-heading --color never -S -g \"**/*\" -e '\\b(func|class|def)\\b' | cut -d: -f1 | sort -u"]}
```

## 7. Шпаргалка по regex для `rg`

* граница слова: `\\bpattern\\b`
* OR: `(foo|bar)`
* без учёта регистра: `-i`
* include/exclude glob: `-g \"**/*.go\" -g \"!vendor/**\"`

## 8. Референс-описание запроса

```text
{
  "command": ["bash", "-lc", "<your command>"],
  "workdir": "./relative/or/absolute/path", // optional
  "timeout_ms": 10000                       // optional
}
```
