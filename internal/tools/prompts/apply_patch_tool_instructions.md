# `apply_patch` — протокол безопасного изменения файлов

## 1. Назначение и границы применения
`apply_patch` используется для внесения правок в файлы через детерминированный патч-формат, пригодный для автоматического применения.

Критическое требование: `apply_patch` вызывается как инструмент. Запрещено пытаться «эмулировать» его через `shell`.

## 2. Каркас патча (обязателен)
Любой патч оформляется единым контейнером:

```text
*** Begin Patch
[ одна или несколько операций над файлами ]
*** End Patch
````

Внутри контейнера размещается одна или несколько операций над файлами.

## 3. Операции над файлами

Каждая операция обязана начинаться с заголовка действия:

* `*** Add File: <path>` — создание нового файла. Все строки содержимого далее обязаны начинаться с `+`.
* `*** Delete File: <path>` — удаление существующего файла. После заголовка никаких строк быть не должно.
* `*** Update File: <path>` — модификация существующего файла (включая возможность переименования).

### 3.1. Переименование при обновлении

Если требуется переименовать файл в рамках `Update`, сразу после строки `*** Update File: ...` добавляется:

```text
*** Move to: <new path>
```

## 4. Фрагменты изменений (hunks) в `Update File`

После `*** Update File: ...` (и опционального `*** Move to: ...`) следует один или несколько фрагментов. Каждый фрагмент начинается с `@@` (допускается заголовок-якорь после `@@`).

Внутри фрагмента каждая строка обязана иметь один из префиксов:

* ` ` (пробел) — контекст, не меняется;
* `-` — строка удаляется;
* `+` — строка добавляется.

## 5. Контекст и «якоря» (как фиксировать место изменения)

Базовый режим:

* показывать 3 строки контекста до изменения и 3 строки после изменения.

Оптимизация:

* если следующее изменение находится в пределах 3 строк от предыдущего, не дублировать конец первого фрагмента как начало второго.

Если 3 строк контекста недостаточно:

* использовать `@@` как привязку и указывать сущность (класс/функцию), в рамках которой производится замена.

Если в пределах функции/класса много похожих блоков и одних якорей недостаточно:

* использовать несколько `@@` подряд, чтобы сузить позиционирование.

### Пример (один якорь)

```text
@@ class BaseClass
[3 строки до]
- [old_code]
+ [new_code]
[3 строки после]
```

### Пример (несколько якорей)

```text
@@ class BaseClass
@@ def method():
[3 строки до]
- [old_code]
+ [new_code]
[3 строки после]
```

## 6. Формальная грамматика (референс)

```text
Patch := Begin { FileOp } End
Begin := "*** Begin Patch" NEWLINE
End := "*** End Patch" NEWLINE
FileOp := AddFile | DeleteFile | UpdateFile
AddFile := "*** Add File: " path NEWLINE { "+" line NEWLINE }
DeleteFile := "*** Delete File: " path NEWLINE
UpdateFile := "*** Update File: " path NEWLINE [ MoveTo ] { Hunk }
MoveTo := "*** Move to: " newPath NEWLINE
Hunk := "@@" [ header ] NEWLINE { HunkLine } [ "*** End of File" NEWLINE ]
HunkLine := (" " | "-" | "+") text NEWLINE
```

## 7. Один патч — несколько операций (допускается)

```text
*** Begin Patch
*** Add File: hello.txt
+Hello world
*** Update File: src/app.py
*** Move to: src/main.py
@@ def greet():
-print("Hi")
+print("Hello, world!")
*** Delete File: obsolete.txt
*** End Patch
```

## 8. Инварианты и запреты

* Заголовок действия (`Add File` / `Delete File` / `Update File`) обязателен для каждой операции.
* В `Add File` все строки содержимого обязаны начинаться с `+`.
* Пути к файлам только относительные. Абсолютные пути запрещены.

## 9. Формат вызова инструмента

```text
apply_patch {"input":"*** Begin Patch\n*** Add File: hello.txt\n+Hello, world!\n*** End Patch\n"}
```
